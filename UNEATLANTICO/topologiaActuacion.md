# Topología (zonas de intervención)

## Fundamentos (I y II Curso)

- **Objetivo**: blindar la adquisición de competencias básicas contra la automatización prematura.
- **Asignaturas clave**: Programación, Estructuras de datos y algoritmos.
- **Riesgo**: Los modelos de IA actuales resuelven trivialmente los ejercicios de estas materias (bucles, arrays, consultas SQL básicas). Si el alumno delega aquí, llegará a III curso sin capacidad de abstracción.
- **Abordaje**: Sala limpia. Prohibición por defecto del uso de generadores de código para la creación de soluciones en entregas evaluables.
- Uso permitido exclusivamente como "Tutor socrático" (para explicar errores, no para solucionarlos).
- Evaluación bajo entornos controlados.

## Transición y herramientas (III Curso)

- El alumno posee ya la base lógica. Se introducen asignaturas donde la IA puede empezar a integrarse como herramienta de productividad.
- **Asignaturas clave**: Ingeniería de software, Programación y sistemas distribuidos, Criptografía
- **Abordaje**: Copilotaje documentado.
- Se permite el uso de IA para tareas mecánicas (boilerplate, generación de datos de prueba).
- **Obligación**: Todo código generado debe estar comentado indicando su origen y validado por el estudiante.

## Simulación profesional (IV Curso)

- **Asignaturas clave**: Inteligencia artificial, Programación web
- **Cambio de paradigma**: En Programación Web y IA, prohibir herramientas de esta naturaleza puede ser contraproducente para la empleabilidad. El alumno debe operar como un Ingeniero Senior.
- **Abordaje**: Responsabilidad editorial.
- Uso libre de herramientas de IA.
- Contrapartida: El alumno es responsable legal y técnico de cada línea. "La IA alucinó" no es una excusa válida para fallos de seguridad o lógica en los desarrollos.
